#### Commit Rules

---

| 타입     | 설명                                                           |
| -------- | -------------------------------------------------------------- |
| feat     | 새로운 기능 추가 (중요한 기능 개발 할 때 마다)                 |
| fix      | 버그 수정                                                      |
| refactor | 코드 리팩토링                                                  |
| style    | 코드 포맷팅, 세미콜론 누락, 코드변경이 없는경우                        |
| test     | 테스트 코드, 리팩토링 테스트 코드 추가                               |
| docs     | 문서 수정 ex) README.md                                        |
| chore    | 빌드 업무 수정, 패키지 매니저 수정                                   |
